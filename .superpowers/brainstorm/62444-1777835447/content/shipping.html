Section 5 — Shipping Flow (Shiprocket)

Rate calculation at checkout · Manual dispatch trigger · Tracking. Respond in terminal.

Two-Phase Shiprocket Usage

Phase A — Rate Fetch (Checkout Step 3)

Customer enters pincode →
shiprocket-rates Edge Function →
Shiprocket Rate Calculator API →
Returns courier options + ETA + price →
Customer picks courier →
Selected rate stored on order
Phase B — Shipment Creation (Admin dispatch)

Admin clicks "Process & Ship" →
create-shipment Edge Function →
Shiprocket Create Order API →
AWB assigned + pickup scheduled →
Tracking ID stored on order →
Customer notified via SMS/email

Products Table — New Shipping Columns

products — add columns
+ weight_kg numeric(5,3) DEFAULT 0.500
+ length_cm numeric(5,1) DEFAULT 30
+ width_cm numeric(5,1) DEFAULT 25
+ height_cm numeric(5,1) DEFAULT 5
Admin sets these per product. Used in rate calculation. Defaults cover typical garment packaging.
