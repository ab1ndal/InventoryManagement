Guest → Account Upgrade

After order confirmation, guest sees:

"Save your details for faster checkout next time?"
[Create account with your phone number]

If they sign up → customers.auth_user_id is set → past order linked to new account automatically via matching email/phone.

Security Principles

  • OTP valid for 60 seconds only, max 3 attempts before cooldown
  • Google OAuth uses Supabase's built-in provider — no custom token handling
  • Supabase RLS policies on cart_items and orders — users can only read/write their own rows
  • Guest session token is a cryptographically random UUID stored in localStorage — not guessable
  • No PII stored in localStorage — only session token + cart variant IDs + quantities
