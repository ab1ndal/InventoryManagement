Customer-Facing: Order Views

/order/:id/confirmation — shown immediately after payment. Order summary, items, total, expected delivery window. "Create account" nudge for guests.
/account/orders — logged-in customer order history. List of past orders with status badge. Requires auth.
/account/orders/:id — single order detail. Items, address, payment method, status timeline, Shiprocket tracking link when dispatched.

pending
Razorpay order created, payment not yet captured
confirmed
Payment verified. Awaiting admin review.
processing
Admin approved. AWB created. Pickup scheduled.
shipped
In transit. Tracking ID active.
delivered
Shiprocket webhook confirms delivery.
cancelled / refunded
Admin action or failed payment.

"Process & Ship" Action Flow

Admin clicks "Process & Ship" →
Confirmation dialog: "Ship #ORD-0042 via [courier name] to [city, pincode]? This will book a pickup."
Admin confirms →
create-shipment Edge Function → AWB assigned →
Order status: processing →
Customer SMS: "Your order #ORD-0042 is being packed and will be picked up shortly."
Once Shiprocket pickup scanned → status: shipped (via webhook) →
Customer SMS: "Your order is on its way! Track: [link]"

Notifications (SMS via Shiprocket built-in)

Shiprocket sends automatic SMS notifications for: order picked up, in transit, out for delivery, delivered.
Platform sends: order confirmed (after payment-verify), order dispatched (after create-shipment).
Email via Resend for: order confirmation (with invoice PDF), dispatch notification.
