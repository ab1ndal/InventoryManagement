New Tables

user_id uuid → auth.users(id)
variant_id uuid → productsizecolors(id)
product_id text → products(productid)
quantity int CHECK > 0
added_at timestamptz DEFAULT now()

UNIQUE (user_id, variant_id) — prevents duplicate rows, uses upsert on add.
Guest cart lives in localStorage; merged into this table on login.

order_items

id uuid PK
order_id uuid → orders(id) ON DELETE CASCADE
variant_id uuid → productsizecolors(id)
product_id text → products(productid)
product_name text (snapshot at time of order)
size text (snapshot)
color text (snapshot)
quantity int
unit_price numeric(10,2) (snapshot)
total_price numeric(10,2)

Why snapshot product fields in order_items?

Product names, prices, sizes can change after an order is placed. Snapshotting at order time ensures order history always shows what the customer actually bought and paid — even if the product is later edited or deleted. This is standard e-commerce practice.
