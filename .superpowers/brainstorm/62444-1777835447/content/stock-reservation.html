Stock Reservation — 15-Minute Hold

Industry-standard pattern. How it works end-to-end.

Flow

Customer clicks "Proceed to Pay"
        │
        ▼
Edge Function: initiate-payment
  1. Check stock availability
     available = stock - SUM(active reservations WHERE expires_at > now())
  2. If available ≥ requested qty:
       INSERT INTO stock_reservations (variant_id, qty, expires_at = now()+15min)
       CREATE Razorpay order → return order_id + key_id to browser
  3. If NOT available → return error "Item out of stock"
        │
        ▼
Browser: Razorpay payment modal opens
  • 15-minute countdown timer shown to customer
  • If timer expires → modal closed, reservation auto-expires
        │
        ▼
Customer completes payment
        │
        ▼
Edge Function: payment-verify
  1. Verify Razorpay signature (HMAC-SHA256)
  2. If valid:
       UPDATE stock in productsizecolors (deduct qty)
       INSERT INTO orders + order_items
       DELETE FROM stock_reservations (cleanup)
  3. If invalid → reject, reservation expires naturally in 15min
        │
        ▼
Order confirmed ✓

New Table: stock_reservations

stock_reservations
├── id            uuid PK
├── variant_id    uuid → productsizecolors.id
├── quantity      int
├── session_token text   (guest session or auth user_id)
├── razorpay_order_id text  (ties reservation to payment attempt)
├── expires_at    timestamptz  DEFAULT now() + '15 minutes'
└── created_at    timestamptz

Available Stock Query (everywhere)

available_stock =
  productsizecolors.stock
  - COALESCE(
      SUM(stock_reservations.quantity)
      WHERE variant_id = $id
        AND expires_at > now()
    , 0)

No cleanup job needed — expired rows are simply invisible to queries. Optionally add a pg_cron job to purge rows older than 1 hour for table hygiene.

UI: Countdown Timer

⏱ 14:32 remaining to complete your purchase
Your items are reserved. Complete payment before the timer expires.

On expiry: timer turns red, modal closes, customer sees "Session expired — items returned to cart."
