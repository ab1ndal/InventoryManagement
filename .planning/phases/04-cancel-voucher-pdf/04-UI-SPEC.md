## Component Inventory

New components required for this phase:

### 1. Cancel Button — BillTable action column (4th action)
- Component: `<Button size="icon" variant="ghost" title="Cancel bill" className="hover:text-destructive">`
- Icon: `<Ban className="h-4 w-4" />` from lucide-react
- Placement: between the PDF icon and the Delete icon in the action `flex gap-1` div
- Disabled states: show Cancel button on Draft and Finalized rows; hide on rows where `paymentstatus === 'cancelled'`
- Tooltip: "Cancel bill" (for Draft); "Cancel finalized bill" (for Finalized)

### 2. CancelConfirmDialog — Step 1 (all bills)
- Component: Shadcn `<Dialog>` (same pattern as BillingForm confirmOpen dialog)
- Trigger: Cancel button click in BillTable
- Title: "Cancel Bill #X?"
- Body: bill details summary — Customer, Date, Grand Total, Status (2–4 rows, `text-sm`)
- Footer actions:
  - Left: `<Button variant="ghost">Keep Bill</Button>`
  - Right (Draft): `<Button variant="destructive">Cancel Draft</Button>` — proceeds directly, no step 2
  - Right (Finalized): `<Button>Continue</Button>` — opens step 2 dialog

### 3. CancelResolutionDialog — Step 2 (Finalized bills with customer only)
- Component: Shadcn `<Dialog>` (second dialog, opens after step 1 closes)
- Title: "How would you like to resolve this?"
- Body: brief explanation — "Bill #X (₹Y) was paid. Choose how to handle the return." + Customer name
- Options (stacked, not inline):
  - Option A button: `<Button variant="outline" className="w-full justify-start">` — label "Return payment to customer" — subtext "Reverses total spend. No store credit issued."
  - Option B button: `<Button className="w-full justify-start">` — label "Issue store credit" — subtext "₹Y added to [Customer Name]'s account. Generates a return receipt."
- Footer: `<Button variant="ghost">Go Back</Button>` — reopens step 1 or dismisses

### 4. Store Credit Badge — BillingForm Summary area
- Placement: inside `<Summary>` component, below "Code Discounts" line, above the GST line
- Condition: render only when `appliedStoreCredit > 0`
- Markup: `<div className="flex justify-between items-center text-green-800 bg-green-50 border border-green-200 rounded px-3 py-1.5 text-sm">`
  - Left: "Store credit applied"
  - Right: `−₹X.XX` + `<button className="ml-2 text-green-600 hover:text-green-900" aria-label="Remove store credit">✕</button>`
- On ✕ click: sets `appliedStoreCredit = 0` in BillingForm state

### 5. Voucher Input — BillingForm
- Placement: below `<DiscountSelector>`, above the Payment section
- Label: `<Label>Promotional Voucher</Label>`
- Input row: `<div className="flex gap-2">`
  - `<Input placeholder="Enter voucher code" value={voucherCode} />`
  - `<Button variant="outline" onClick={handleApplyVoucher}>Apply Voucher</Button>`
- States:
  - Idle: input + Apply button as described
  - Loading: Apply button shows `<Loader2 className="h-4 w-4 animate-spin" />`, disabled
  - Applied: input row replaced by voucher badge (see below); input is hidden
  - Error: input remains visible; error text `<p className="text-xs text-destructive mt-1">{errorMessage}</p>` below input
- Applied voucher badge: `<div className="flex justify-between items-center bg-blue-50 border border-blue-200 text-blue-800 rounded px-3 py-1.5 text-sm">`
  - Left: "Voucher #{code}: ₹Y applied"
  - Right: `<button className="ml-2 text-blue-600 hover:text-blue-900" aria-label="Remove voucher">✕</button>`

### 6. ReturnReceiptView — off-screen PDF render component
- Pattern: identical to InvoiceView.js (forwardRef, off-screen fixed div, inline styles)
- Container: `width: "400px"` (compact, not A4 794px) — single column layout
- Sections (top to bottom):
  1. Store header: logo (height: 60px) + STORE.name (fontSize: 14px, fontWeight: 600, letterSpacing: 1px) + STORE.address + STORE.phone — horizontal flex, borderBottom
  2. Receipt label: `"STORE CREDIT RECEIPT"` — fontSize: 13px, fontWeight: 600, textAlign: center, textTransform: uppercase, letterSpacing: 2px, padding: "8px 0", borderBottom
  3. Bill details: Bill #, Original date, Customer name — two-column key/value, fontSize: 11px
  4. Divider line (borderTop: "1px solid #e5e7eb")
  5. Items cancelled: compact table — Name, Qty, MRP columns — fontSize: 11px
  6. Divider line
  7. Credit amount row: "Store Credit Issued: ₹X" — fontWeight: 600, fontSize: 13px, textAlign: right
  8. Issue date: fontSize: 10px, color: "#6b7280", textAlign: right
  9. Note text: "Store credit has been added to your account and will be automatically applied on your next purchase." — fontSize: 10px, color: "#6b7280", marginTop: 8px, textAlign: center
- STORE constants: import from InvoiceView.js or copy the same object

## Interaction States

### Cancel Button in BillTable

| Bill Status | Cancel button visible? | Behavior |
|-------------|----------------------|----------|
| Draft | Yes | Opens CancelConfirmDialog (step 1); on confirm → silent cancel + toast |
| Finalized, no customer | Yes | Opens CancelConfirmDialog (step 1); on confirm → cancel + toast, skip step 2 |
| Finalized, with customer | Yes | Opens CancelConfirmDialog (step 1); on confirm → opens CancelResolutionDialog (step 2) |
| Cancelled | No (hide button) | No action available |

### Store Credit Auto-Apply

| Condition | UI |
|-----------|-----|
| Customer selected, store_credit = 0 | No badge rendered |
| Customer selected, store_credit > 0 | Green badge auto-renders in Summary |
| Staff removes badge (✕) | Badge disappears, appliedStoreCredit = 0 |
| Customer changes | Badge resets — re-query new customer's store_credit |
| Bill reloads for edit | Re-query customer store_credit on load |

### Voucher Input

| State | UI |
|-------|-----|
| No input | Input field + Apply button (outline, disabled if input empty) |
| Typing | Apply button enabled |
| Apply clicked | Button shows spinner, input disabled |
| Valid voucher | Input row hidden; applied badge rendered |
| Invalid voucher | Input visible; error text below input; Apply re-enabled |
| Staff removes badge (✕) | Badge disappears; input row restored empty |

## Visual Focal Points

Primary visual anchors per affected screen:

- **BillingForm:** Grand Total row in the Summary panel — largest bold number, border-top separator, always in view during billing. New store credit (green) and voucher (blue) badge rows draw eye upward to the deduction chain.
- **BillTable:** Status badge column + action column — the "Cancelled" destructive badge is the primary state indicator; the action column (Edit/PDF/Cancel/Delete icons) is the interaction target.

---

## Deduction Order in Summary Display

The Summary component must display deductions in this exact order:

1. Subtotal (MRP + Alterations)
2. Item Discounts (red) — existing
3. Code Discounts (red) — existing
4. Promo Voucher (blue badge) — new; shows "−₹Y" when applied
5. Store Credit (green badge) — new; shows "−₹X" when applied
6. GST — existing
7. **Grand Total** (bold, border-top) — after all deductions, floored at 0

The Summary component receives `appliedStoreCredit` and `appliedVoucher` as props (or reads from context). It renders the new badge rows conditionally.
