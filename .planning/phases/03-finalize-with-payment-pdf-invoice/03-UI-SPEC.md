## Interaction Contracts

### 1. Finalize Button (BillingForm footer)

State machine:

| State | Visual | Trigger |
|-------|--------|---------|
| Default | Button label "Finalize", primary color, enabled | — |
| Missing customer | Button click → inline validation toast fires | customer not selected |
| Missing payment | Button click → inline validation toast fires | paymentMethod or paymentAmount empty |
| Amount mismatch | Button click → inline validation toast fires | abs(paid - grandTotal) > 100 |
| All valid | Confirmation dialog opens | passes all validation |
| Saving | Button disabled, label "Saving..." | during DB + PDF sequence |
| Success | Dialog closes, BillTable refreshes, PDF opens in new tab | onSubmit?.()+onOpenChange?.(false) |

### 2. Confirmation Dialog

Triggered: after all inline validations pass on "Finalize" click.

Content to display:
- DialogTitle: "Confirm Finalize"
- DialogDescription: "This action cannot be undone."
- Bill # (billId)
- Customer name (resolved from selectedCustomerId)
- Grand Total: ₹{grandTotal} (2 decimal places, Indian comma formatting)
- Payment Method: {cash | card | upi | mixed} — displayed as capitalized text
- Amount Received: ₹{paymentAmount}
- "Confirm & Finalize" button — primary, full-width or right-aligned
- "Cancel" button — ghost variant, left of confirm

Interaction sequence on "Confirm & Finalize":
1. `setIsSaving(true)` — disables both buttons
2. UPDATE bills (finalized=true, paymentstatus='finalized', payment_method, payment_amount, finalized_at=now)
3. UPDATE customers (total_spend += grandTotal, last_purchased_at = today)
4. INSERT discount_usage rows for each selectedCode (customerid NOT NULL — always safe per D-06/D-07)
5. Generate PDF (html2canvas on invoiceRef.current → jsPDF blob)
6. Upload blob to Supabase Storage `invoices/bill-{billid}.pdf` (upsert: true)
7. UPDATE bills SET pdf_url = getPublicUrl result
8. `window.open(pdfUrl, '_blank')` — opens invoice in new tab
9. Toast: `{ title: "Bill #${billId} finalized" }` (success, no variant)
10. `onOpenChange?.(false)` + `onSubmit?.()` to close and refresh

Error handling:
- DB failure (steps 2–4): destructive toast, `setIsSaving(false)`, dialog stays open, bill NOT finalized
- PDF generation failure (step 5): destructive toast "Invoice PDF failed to generate. Bill is finalized — reprint from Bill List.", `setIsSaving(false)`, dialog closes (bill DB state already committed per CONTEXT.md D-01 discretion)
- Storage upload failure (step 6–7): same as PDF generation failure — bill committed, PDF can be regenerated from BillTable

### 3. BillTable Print Icon

Condition: show FileText icon only when `row.finalized === true` AND `row.pdf_url` is non-null.

| State | Visual |
|-------|--------|
| finalized + pdf_url present | FileText icon, primary color (#0066cc), cursor-pointer, tooltip "Print Invoice" |
| finalized + pdf_url null | FileText icon, muted color (#9ca3af), cursor-not-allowed, tooltip "PDF not yet generated" |
| not finalized | No print icon shown |

Click action: `window.open(row.pdf_url, '_blank')` (no confirmation needed — read-only action).

### 4. Customer Validation Toast (existing pattern, extended)

Trigger: "Finalize" clicked with no customer selected.

Toast content: `{ title: "Customer required", description: "A customer must be selected before finalizing.", variant: "destructive" }`

Source: D-06 exact copy from CONTEXT.md.

## InvoiceView Layout Contract

The InvoiceView component renders an A4-proportioned invoice for html2canvas capture. It is never shown to the user — only captured as a bitmap.

### Dimensions
- Width: 794px (fixed, A4 at 96dpi)
- Height: auto (content-driven; jsPDF scales to fit)
- Background: #ffffff

### Sections (top to bottom)

1. **Store Header** (center-aligned)
   - Store name: "BINDAL'S CREATION" — 20px bold, letter-spacing wide, all-caps
   - Tagline: "A COMPLETE RANGE OF FAMILY WEAR" — 12px regular, muted gray
   - Address: "58 Sihani Gate Market, Ghaziabad 201001" — 11px regular
   - Phone: "+91 9810873280 | +91 9810121438" — 11px regular
   - GSTIN: "GSTIN: 09ABVPB4203A1Z4" — 11px semibold (required for GST compliance)
   - Divider line: 1px solid #e5e7eb

2. **Bill Metadata** (two-column layout, left and right)
   - Left: Bill No: {billId}, Date: {DD/MM/YYYY}
   - Right: Customer: {name}, Salesperson(s): {comma-separated names}
   - Font: 11px regular, labels at weight 600

3. **Line Items Table**
   - Columns: S.No. | Particulars | Size | Color | Qty | Rate (MRP) | Disc | GST% | Taxable | CGST | SGST | Amount
   - Header row: 11px weight 600, background #f4f4f5
   - Data rows: 11px weight 400, alternating white/#fafafa
   - "Particulars" = product_name (manual items show name; inventory items show name)
   - Rate = MRP (before discount)
   - Disc = item-level discount amount (₹)
   - Taxable = (MRP - disc) / (1 + gst_rate/100) — taxable value excl. GST
   - CGST = taxable * (gst_rate/2) / 100
   - SGST = taxable * (gst_rate/2) / 100
   - Amount = MRP - disc (inclusive, what customer pays for this line)

4. **Totals Section** (right-aligned)
   - Item Subtotal (sum of Amount column)
   - Overall Discount: -{₹overallDiscount} (if any codes applied, show code names)
   - Grand Total: ₹{grandTotal} — 14px bold
   - GST Summary: Total CGST: ₹X | Total SGST: ₹Y

5. **Payment Footer**
   - Payment Method: {cash / card / upi / mixed}
   - Amount Received: ₹{paymentAmount}
   - Divider

6. **Notes / Footer**
   - "Note: Goods once sold cannot be taken back." (from sample_bill.jpg)
   - "Auth Signature" right-aligned (blank line for physical signature if printed)

Source: sample_bill.jpg visual reference; D-04/D-05 constants from CONTEXT.md; PRINT-01 requirements; Indian GST CGST+SGST split per CONTEXT.md specifics section.
