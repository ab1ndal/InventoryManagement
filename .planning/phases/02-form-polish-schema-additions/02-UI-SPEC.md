## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding (e.g. badge px) |
| sm | 8px | Compact element spacing, gap between form fields in a row |
| md | 16px | Default element spacing, card padding, dialog body padding |
| lg | 24px | Section padding, gap between form sections |
| xl | 32px | Layout gaps, dialog footer spacing |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets for icon buttons (Edit, Delete, PDF placeholders in BillTable action column): minimum 36px × 36px rendered area (use `size="icon"` Button variant which renders at h-9 w-9 = 36px)
- Chip dismiss button (×) inside salesperson chips: 20px × 20px with 4px padding (accessible enough for admin-only desktop UI)

### New patterns to implement

#### Salesperson combobox (UI-03)
Pattern: Input-triggered dropdown with checkboxes. Model after `CustomerSelector.js` (uses `bg-white border rounded shadow` dropdown div with `hover:bg-gray-100` items). Replace single-select with checkboxes.

Structure:
```
[Search input — placeholder "Search salesperson"]
[Dropdown div — absolute, z-10, bg-white, border, rounded, shadow]
  [Checkbox + Label per salesperson row — px-3 py-2 hover:bg-gray-100]
[Selected chips row — flex flex-wrap gap-1 mt-1]
  [Chip — inline-flex items-center gap-1 bg-secondary text-secondary-foreground rounded-full px-2 py-0.5 text-xs]
    [Name text]
    [× dismiss button — text-muted-foreground hover:text-foreground]
```

Chip visual: `bg-secondary` (#f4f4f5) with `text-secondary-foreground` (#1f2937), rounded-full, text-xs (12px), font-normal. Dismiss × is a plain `<button>` with `ml-1 opacity-60 hover:opacity-100`.

Keyboard: dropdown closes on Escape; checkbox toggles on Space/Enter. Focus traps not required (admin-only, desktop).

#### ManualItemForm grouped sections (UI-02)
Two labeled sections using visual separation (a `text-xs font-semibold text-muted-foreground uppercase tracking-wide` section header, not a Card border):

**Item Details** — grid-cols-2 gap-2:
- Category (Input, placeholder "e.g. Shirt")
- Product Name (Input, placeholder "Item name", required — show red asterisk in label)
- Product Code (Input, placeholder "Optional", optional)
- Size (Input, placeholder "e.g. L, 42", free-text)
- Color (Input, placeholder "e.g. Navy Blue", free-text)

**Pricing** — grid-cols-2 gap-2:
- Qty (Input type=number, min=1, placeholder "1")
- MRP ₹ (Input type=number, placeholder "0.00")
- Alteration Charge ₹ (Input type=number, placeholder "0.00", default 0)
- GST Rate % (Select: options 0%, 5%, 12%, 18%, 28%; default 18%)
- Z Code ₹ (Input type=number, placeholder "Cost price", helper text below: "Internal only — not shown to customer")

#### BillTable action column icons (D-14 through D-17)
Action column renders three icon buttons in a `flex gap-1 justify-center` container:

| Icon | Lucide component | State in Phase 2 | Props |
|------|-----------------|-----------------|-------|
| Edit | `Pencil` (16px) | Active — calls `onEdit(b.billid)` | `size="icon"` variant="ghost" |
| PDF | `FileText` (16px) | Disabled placeholder | `size="icon"` variant="ghost" `disabled` `opacity-40` `cursor-not-allowed` `title="Available after finalize"` |
| Delete | `Trash2` (16px) | Disabled placeholder | `size="icon"` variant="ghost" `disabled` `opacity-40` `cursor-not-allowed` `title="Available in Phase 4"` |

Customer name column: add after Bill ID column. Display `{first_name} {last_name}` from customers JOIN; fallback `—` if no customer. Requires updating the Supabase select query to JOIN customers.

Monetary value display: all ₹ values use `.toFixed(2)` — already present in BillTable for totalamount, gst_total, discount_total. Apply same to any newly added columns.
