## Architecture

**React 19 SPA** using Create React App, backed by **Supabase** (PostgreSQL + Auth). No server-side code — all data access is direct from the client via `@supabase/supabase-js`.

### Routing

All routes live in `src/App.js`. The structure is:
- `/` → redirects to `/admin/inventory`
- `/admin/*` → protected by `RequireAdminAuth` wrapper, rendered inside `AdminLayout`
- Public auth pages: `/admin/login`, `/admin/signup`, `/admin/forgot-password`, `/admin/reset-password`
- `/unauthorized` → shown when a user lacks the required role

### Auth & Roles

`src/admin/components/RequireAdminAuth.jsx` guards all admin routes. It:
1. Calls `supabase.auth.getSession()` on mount
2. Queries the `profiles` table for the user's role
3. Allows access only if role is `"admin"` or `"superadmin"`
4. Listens to `supabase.auth.onAuthStateChange` for real-time session updates

### Data Layer

`src/lib/supabaseClient.js` initializes the Supabase client using env vars:
- `REACT_APP_SUPABASE_URL`
- `REACT_APP_SUPABASE_ANON_KEY`

Copy `.env.example` to `.env` and fill in these values before running locally.

All database access goes through the Supabase client directly in component effects/handlers — no separate API layer or Redux.

### Key Database Tables

| Table | Purpose |
|-------|---------|
| `products` | Inventory items |
| `productsizecolors` | Size/color variants per product |
| `categories` | Product categories |
| `customers` | Customer records |
| `suppliers` | Supplier records |
| `profiles` | User roles (`admin` / `superadmin`) |
| `discounts` | Discount codes |
| `bills` | Invoices/billing records |
| `mockups` | Product mockup tracking |

**Product ID format:** `BC{YY}{###}` (e.g., `BC25001`). New IDs are generated by calling the Supabase RPC `get_max_product_suffix(p_prefix)`.

### UI Stack

- **Shadcn/ui** components in `src/components/ui/` (Radix UI primitives + Tailwind)
- **Tailwind CSS** with custom theme in `tailwind.config.js` (primary: `#0066cc`)
- **Sonner** for toast notifications
- **React Hook Form + Zod** for form validation
- **Plotly.js** (`react-plotly.js`) for charts/graphs on MockupGraphsPage and TotalValuePage
- Path alias `@/*` maps to `src/*`

### State Management

No global store. Each page/component manages its own state with `useState`/`useEffect`. A common pattern for triggering re-fetches is a boolean `refreshFlag` that gets toggled after mutations.

### Printing

`qz-tray` is used for label/invoice printing. It requires the QZ Tray desktop application to be running on the user's machine.

### Code Style

- Admin pages: `src/admin/pages/` — full page components
- Admin UI components: `src/admin/components/` (with `billing/` and `archive/` subdirectories)
- Utility functions: `src/utility/` (`dateFormat.js`, `formatPhone.js`)
- Mix of `.js`, `.jsx`, and `.ts`/`.tsx` extensions — all are valid React components

## Database Schema (Live)

Before writing any migration or DB query, fetch live schema from Supabase:

```bash
source .env && curl -s \
  -H "apikey: $SUPABASE_SERVICE_ROLE_KEY" \
  -H "Authorization: Bearer $SUPABASE_SERVICE_ROLE_KEY" \
  "$REACT_APP_SUPABASE_URL/rest/v1/rpc/get_schema_info" | \
  node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); const rows=JSON.parse(d); const tables={}; rows.forEach(r=>{if(!tables[r.table_name])tables[r.table_name]=[]; tables[r.table_name].push(r.column_name+' '+r.udt_name+(r.is_nullable==='NO'?' NOT NULL':'')+(r.column_default?' DEFAULT '+r.column_default:'')); }); Object.entries(tables).forEach(([t,cols])=>console.log(t+':\n  '+cols.join('\n  ')));"
```

Use the output (not `schema/*.sql` files) as ground truth for current column names and types. If the RPC is missing, apply `schema/util_get_schema_info.sql` once in the Supabase SQL editor first.

New migrations go in `schema/migration_*.sql`. Never edit existing `schema/*.sql` table files.
